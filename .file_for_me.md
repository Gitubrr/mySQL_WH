## 0. Connection to the server
```bash
mysql -u root -p                      # Login as superuser
mysql -u username -p                  # Login as a specific user
mysql -h host_address -u username -p  # Connecting to a remote server
```

```sql
help;                               -- \h

clear;                              -- \c  Cancel current entry

```

## 2. DDL (Data Definition Language)
Creation and modification of structure (databases, tables, indexes).

### Databases
```sql
SHOW DATABASES;                     -- List of all databases
CREATE DATABASE db_name;            -- Create a database
USE db_name;                        -- Select a base to work with
SELECT DATABASE();                  -- Find out which base is currently selected
DROP DATABASE IF EXISTS db_name;    -- Delete database
```

### Tables
```sql
SHOW TABLES;                        -- List of tables in the selected database
DESCRIBE table_name;                -- Table structure (columns, types)
DROP TABLE IF EXISTS table_name;    -- Delete table
ALTER TABLE tab ADD col INT;        -- Add a column to the table
```

## 3. DML & DQL (Data Manipulation / Query)
Working with table contents (rows, cols).\
**DQL** - Only retrieves data.\
**DML** - Changes data.
### DQL (Data Query Language)

### Selection
Used to filter rows based on conditions.
```sql
SELECT * FROM table_name WHERE cond

SELECT * FROM Pet WHERE Age > 10;
SELECT * FROM Pet WHERE Age BETWEEN 2 AND 7;
SELECT * FROM Pet WHERE Pet_Type_ID IN (1, 2);

SELECT * FROM Person WHERE Address LIKE '%VO%';
SELECT * FROM Order1 WHERE Is_Done = 0 AND Mark IS NULL;
SELECT * FROM Pet WHERE Pet_Type_ID > 2 AND NOT Nick = 'Musia';

SELECT * FROM Person 
WHERE Person_ID IN (SELECT Person_ID FROM Owner);

SELECT * FROM Order1 
WHERE Pet_ID IN (SELECT Pet_ID FROM Pet WHERE Age > (SELECT AVG(Age) FROM Pet));

SELECT * FROM Order1 WHERE HOUR(Time_Order) = 9;
SELECT * FROM Person WHERE LENGTH(First_Name) < 5;
```

### Projection
Used to choose specific columns from the table.
```sql
SELECT cols FROM table_name

SELECT Last_Name FROM Person;
SELECT Nick, Breed, Age FROM Pet;

SELECT DISTINCT Pet_Type_ID FROM Pet;   -- unique only
SELECT DISTINCT Last_Name FROM Person;

SELECT Nick AS Name, Breed FROM Pet;
SELECT Phone AS Contact_Number FROM Person;
SELECT Nick, Age, (Age * 7) AS Human_Age FROM Pet;

SELECT Order_ID, HOUR(Time_Order) AS Appointment_Hour FROM Order1;
SELECT CONCAT(First_Name, ' ', Last_Name) AS Full_Name FROM Person;

SELECT AVG(Age), MIN(Age), MAX(Age) FROM Pet;
```

### Join 
Combining two tables into one based on a common attribute (Foreign Key).
```sql
SELECT * FROM table1 JOIN table2 ON t1.id = t2.fk

SELECT Person.First_Name, Person.Last_Name, Owner.Description
FROM Person
JOIN Owner ON Person.Person_ID = Owner.Person_ID;

SELECT Person.Last_Name, Pet.Nick
FROM Person
LEFT JOIN Owner ON Person.Person_ID = Owner.Person_ID
LEFT JOIN Pet ON Owner.Owner_ID = Pet.Owner_ID;
```

### Union 
Combines the results of two queries and removes duplicates by default.\

```sql
query1 UNION query2

SELECT First_Name FROM Person JOIN Employee ON Person.Person_ID = Employee.Person_ID
UNION
SELECT First_Name FROM Person JOIN Owner ON Person.Person_ID = Owner.Person_ID;
-- Get a single list of all unique first names from both Employees and Owners.
```

### Intersection
Returns only the rows that are present in both result sets.
```sql
query1 INTERSECT query2

SELECT Person_ID FROM Employee
INTERSECT
SELECT Person_ID FROM Owner;
-- Find IDs of people who are both an Employee and an Owner.
```

### Difference 
Returns rows from the first query that are not present in the second.
```sql
query1 EXCEPT query2

SELECT Person_ID FROM Employee
EXCEPT
SELECT Person_ID FROM Owner;
-- Find IDs of people who are Employees but not Owners.s
```

### Cartesian Product 
Pairing every row of the first table with every row of the second.
```sql
SELECT cols FROM t1 CROSS JOIN t2

SELECT 
    Person.First_Name AS Employee, 
    Service.Name AS Service_Name
FROM Employee
JOIN Person ON Employee.Person_ID = Person.Person_ID
CROSS JOIN Service;
-- Employee × Service
```


### DML (Data Manipulation Language)

### Insertion 
Adds new data into a table (Adding Rows).
```sql
INSERT INTO table_name (cols) VALUES (values)

INSERT INTO Pet (Pet_ID, Nick, Pet_Type_ID, Owner_ID) 
VALUES (11, 'Rex', 1, 3);
```

### Update 
Changes existing data (Modifying Rows).
```sql
UPDATE table_name SET col = value WHERE cond

UPDATE Pet 
SET Age = Age + 1 
WHERE Nick = 'Rex';
```

### Deletion 
Removes data from a table (Removing Rows).
```sql
DELETE FROM table_name WHERE cond

DELETE FROM Order1 
WHERE Is_Done = 1 AND Time_Order < '2023-01-01';
```


## 4. DCL (Data Control Language)
Commands for access and security management.
```sql
CREATE USER 'u'@'localhost' IDENTIFIED BY 'p';  -- Create a user
GRANT ALL ON db.* TO 'u'@'localhost';           -- Give full rights to the database
REVOKE UPDATE ON db.* FROM 'u'@'localhost';     -- Take away the right to update
FLUSH PRIVILEGES;                               -- Update the permissions table
```

## 5. TCL (Transaction Control Language)
Transaction management (rollback of changes).
```sql
SET autocommit = 0;                 -- Turn off autosave
COMMIT;                             -- Save changes permanently
ROLLBACK;                           -- Roll back changes
```
